api error codes

a guide to the error codes returned by the sentinelpay api.

error codes

all api errors return a standard http status code. the response body is json and includes a human-readable error field where applicable.

400

bad request

missing or malformed wallet field. the address must be 0x-prefixed and exactly 42 characters long.

401

unauthorized

missing or invalid x-api-key header. check that your key is active and correctly copied.

402

payment required

your credit balance has reached zero. purchase additional credits from the dashboard.

403

forbidden

account email is unverified, or the public endpoint scan limit has been reached for this ip or browser fingerprint.

413

payload too large

request body exceeds the allowed size.

429

too many requests

you have exceeded the rate limit of 30 requests per 15 minutes. wait before retrying.

500

internal server error

unexpected server-side error. if persistent, contact support with the timestamp of the request.

503

service unavailable

the upstream blockchain data provider returned an error. retry after a short delay.

504

gateway timeout

the scoring engine did not respond within 25 seconds. this can happen for wallets with very high transaction counts. retry once.

credits are not deducted on failed requests — only successful 200 responses consume a credit.

handling errors in production

for 503 and 504 responses, implement exponential backoff with a maximum of 3 attempts. for 429, read the retryAfter field from the response body and wait that many seconds before retrying:

if (res.status === 429) {
  const { retryAfter } = await res.json();
  await sleep(retryAfter * 1000);
  // retry request
}

do not retry 400, 401, 402, or 403 errors automatically — these require a fix on the client side before retrying.
