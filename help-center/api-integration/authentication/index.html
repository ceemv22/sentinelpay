authentication

all sentinelpay api requests must be authenticated using an api key passed in the request header.

your api key

an api key is automatically provisioned for your organization the moment it is created. the full raw key is displayed once at creation time — copy it and store it securely in your environment variables or secrets manager. after that, only the last four characters are visible in the dashboard header as a reference.

api keys are secrets. never commit them to source control, log them, or expose them in client-side code. treat them with the same sensitivity as passwords.

using your key

pass your api key as a header on every request to authenticated endpoints:

x-api-key: your_api_key_here

requests missing this header — or using an invalid or deactivated key — will receive a 401 unauthorized response.

rolling your key

if your key is compromised or needs rotation, navigate to your organization settings tab and use the roll key option. rolling immediately deactivates the old key and issues a new one. any service still using the old key will receive 401 responses until updated.

only the organization owner can roll the api key. developer-role members do not have access to this action.
