request payloads

understand the json structure for requests and responses.

request body

{
  "wallet": "0xAbCd1234..."
}
field type required description
wallet string yes ethereum wallet address. must start with 0x and be exactly 42 characters total.

response body

{
  "score": 50,
  "category": "medium",
  "flags": ["new_wallet", "high_velocity"],
  "history_incomplete": false,
  "timestamp": "2026-06-01T10:00:00.000Z"
}
field type description
score integer risk score from 0 (clean) to 100 (highest risk). sanctioned wallets always return exactly 100.
category string "low" (0–29), "medium" (30–59), or "high" (60–100).
flags string[] list of triggered risk heuristics. empty array means no signals were detected.
history_incomplete boolean true if any transaction list hit the 10,000 tx fetch limit. the score reflects only what was analyzed, not the wallet's full history.
timestamp string (ISO 8601) utc timestamp of when the scan completed.

flag values

the flags array can contain any combination of the following string values:

  • sanctioned_entity
  • mixer_interaction
  • new_wallet
  • high_velocity
  • io_imbalance

see specific risk flags for what each value means and how to interpret it.
