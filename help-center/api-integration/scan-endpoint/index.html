the scan endpoint

learn how to interact with the /v1/score endpoint to assess wallet risk.

endpoint overview

the scan endpoint takes an ethereum wallet address and returns a risk score with full flag detail. it is synchronous — the result is returned in the same http response, typically within 3–25 seconds depending on wallet history size.

POST https://sentinelpay.org/v1/score

all requests require a valid api key in the x-api-key header. see authentication for key management and rotation.

request

send a Content-Type: application/json body with one required field:

{
  "wallet": "0xAbCd1234..."
}
  • wallet (string, required) — an ethereum wallet address. must be 0x-prefixed and exactly 42 characters long.

response

a successful scan returns http 200 with the following json body:

{
  "score": 70,
  "category": "high",
  "flags": ["mixer_interaction", "high_velocity"],
  "history_incomplete": false,
  "timestamp": "2026-06-01T10:00:00.000Z"
}
score

integer (0–100)

the calculated risk score. higher is riskier.

category

string

low (0–29), medium (30–59), or high (60–100).

flags

string[]

heuristics that fired. empty array if none triggered.

history_incomplete

boolean

true if the wallet's history exceeded 10,000 entries and was only partially analyzed.

timestamp

ISO 8601 string

utc datetime of when the scan was performed.

example

curl -X POST https://sentinelpay.org/v1/score \
  -H "Content-Type: application/json" \
  -H "x-api-key: your_api_key_here" \
  -d '{"wallet": "0xAbCd1234..."}'

rate limit

api key requests are limited to 30 per 15 minutes. exceeding this returns 429 too many requests. see api rate limits for the full limits table.
