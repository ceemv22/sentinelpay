Introduction to SentinelPay

SentinelPay is an institutional-grade, pre-deposit risk oracle designed to intercept illicit crypto flows before they contaminate your treasury.

Built for crypto casinos, OTC desks, and high-risk merchants, our engine evaluates incoming wallet addresses and transactions in real-time. By leveraging advanced heuristics, ML models, and extensive proprietary databases, we provide an immediate risk score (0-100) allowing you to automate compliance and reject tainted funds instantly.

Note for MVP Users: The current platform supports Ethereum (ERC-20) addresses. Bitcoin and Solana support is rolling out in Q3.

Quickstart Guide

Integrate SentinelPay into your deposit flow in three simple steps.

1. Create an Organization

Navigate to your dashboard and create an institutional workspace. This isolated environment will house your API keys and scan history.

2. Generate API Keys

Inside your organization's dashboard, locate the API key widget in the header. Reveal your key and store it securely. We do not store raw keys on our servers.

3. Make your first API Call

Use your generated `sp_live_...` token to query the risk engine. See the Scan Endpoint section for a complete payload example.

Scan Endpoint

The primary endpoint for checking an address's risk profile before processing a deposit.

GET https://api.sentinelpay.org/v1/scan/:address

Request Headers

Header Required Description
Authorization Yes Bearer token using your `sp_live_...` API key.
Content-Type Yes Must be application/json

Example Response
