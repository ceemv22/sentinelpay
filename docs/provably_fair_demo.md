# provably fair // technical implementation

a demonstration of a secure, transparent, and non-manipulatable fairness system for igaming platforms.

## core logic

the system utilizes a three-part hash chain to ensure that neither the house nor the player can predict or manipulate the outcome of a game.

1.  **server seed**: a cryptographically secure random string generated by the server. it is hashed (sha256) and shown to the player *before* the bet.
2.  **client seed**: a random string provided by the player (or their browser).
3.  **nonce**: a counter that increments with each bet made using the current seed pair.

## algorithm overview

```javascript
const crypto = require('crypto');

/**
 * generates a game outcome based on seeds and nonce.
 * @param {string} serverSeed - the raw secret server seed.
 * @param {string} clientSeed - the player-provided seed.
 * @param {number} nonce - the current bet number.
 * @returns {number} - a result between 0 and 1,000,000.
 */
function generateOutcome(serverSeed, clientSeed, nonce) {
    const combined = `${serverSeed}:${clientSeed}:${nonce}`;
    const hash = crypto.createHmac('sha256', serverSeed)
                       .update(`${clientSeed}:${nonce}`)
                       .digest('hex');

    // use the first 8 characters of the hash for the result
    const resultInt = parseInt(hash.substring(0, 8), 16);
    return resultInt % 1000001;
}

// example usage for a coin flip (50/50)
const outcome = generateOutcome("secret_server_seed", "player_choice", 1);
const isHeads = outcome < 500000;

console.log(`outcome: ${outcome}, result: ${isHeads ? 'heads' : 'tails'}`);
```

## verification process

players can verify the fairness of any past game by:
- taking the revealed raw server seed (after they rotate seeds).
- hashing it and comparing it to the pre-bet hash they were shown.
- re-running the outcome algorithm with their client seed and the nonce.

if the results match, the game is proven to be fair.
